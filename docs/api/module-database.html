Module: database

Database Module - JSON File Operations Handles all data persistence operations for the TSV Ledger application. Manages expenditure data storage and retrieval using JSON files. Supports test database isolation through environment variables.

(inner) addExpenditure(expenditure, callback)

Adds a new expenditure to the data store Creates a new expenditure record with auto-generated ID and persists it to storage. Follows REST API patterns for resource creation.
Parameters:
Name Type Description
expenditure Object New expenditure data (without id)
Properties
Name Type Attributes Description
description string Description of the expense (required)
amount number Expense amount (required)
date string <optional>
 Date of the expense (optional)
category string <optional>
 Business category (optional)
callback function Callback function with (error, data) signature
Properties
Name Type Description
error Error | null Error object if operation failed, null on success
data Object Created expenditure object with generated id
Source:
  • database.js, line 153
Example
const newExpense = {
  description: 'Team lunch at Texas Roadhouse',
  amount: 200.00,
  date: '2025-01-16',
  category: 'Food & Dining'
};

addExpenditure(newExpense, (error, createdExpense) => {
  if (error) {
    console.error('Failed to add expenditure:', error);
    return;
  }
  console.log('Created expense with ID:', createdExpense.id);
});
